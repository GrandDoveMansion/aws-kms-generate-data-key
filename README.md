## ❓ Quick FAQ

**Is this really free?** The tool is free. AWS may charge for KMS usage (first 20,000 requests/month free).

**What is a data key?** A symmetric key used for client-side encryption of data (like file encryption, database encryption).

**Why use AWS KMS for data keys?** Keys are generated securely within AWS KMS, never exposed unnecessarily.

**What's the difference between plaintext and encrypted key?** Plaintext key is used for actual encryption. Encrypted key is stored alongside encrypted data for later decryption.

**What is encryption context?** Additional authenticated data that must match during decryption (key-value pairs).

**What AES mode should I use?** AES_256 is recommended for high-security applications. AES_128 is sufficient for most use cases.

**Can I reuse a data key?** Yes, but best practice is to generate a new key for each data item.

**Do I need a specific KMS key?** Yes — a symmetric encryption KMS key (type: SYMMETRIC_DEFAULT).

**Is this safe to use?** Yes — no malware, uses official AWS SDK, data keys securely generated by AWS KMS.

**What if key generation fails?** Check IAM permissions and KMS key status.
